Vehicle Rental System
Lab 8 (Meeting 8): Domain model setup: Vehicle (ID + nextId), concrete types, interfaces (Parkable, CombustionVehicle), initial toString().
Lab 9 (Meeting 9): Garage + parking/unparking consistency (garage ↔ vehicle) + menu operations: add/remove/print (with clear messages).

Lab 11 (Meeting 11): XML persistence: load on startup + save on exit; fuelType mapping; full integration and stability testing.

Description

This Java application implements a vehicle rental system based on object-oriented programming concepts.

The system uses abstract classes, interfaces, inheritance, polymorphism, collections, sorting algorithms, and XML persistence.

Vehicles can be added, removed, sorted, parked in garages, loaded from XML files, and saved automatically through a console-based menu interface.
